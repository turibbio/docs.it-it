# <a name="structures-and-classes-visual-basic"></a><span data-ttu-id="02674-102">Strutture e classi (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="02674-102">Structures and Classes (Visual Basic)</span></span>

 <span data-ttu-id="02674-105">Le classi hanno il vantaggio di essere tipi di riferimento. il passaggio di un riferimento è più efficiente rispetto al passaggio di una variabile di struttura con tutti i relativi dati.</span><span class="sxs-lookup"><span data-stu-id="02674-105">Classes have the advantage of being reference types — passing a reference is more efficient than passing a structure variable with all its data.</span></span> <span data-ttu-id="02674-106">D'altra parte, le strutture non richiedono l'allocazione della memoria nell'heap globale.</span><span class="sxs-lookup"><span data-stu-id="02674-106">On the other hand, structures do not require allocation of memory on the global heap.</span></span>

## <a name="similarities"></a><span data-ttu-id="02674-109">Somiglianze</span><span class="sxs-lookup"><span data-stu-id="02674-109">Similarities</span></span>  
 <span data-ttu-id="02674-110">Le strutture e le classi sono simili nei seguenti aspetti:</span><span class="sxs-lookup"><span data-stu-id="02674-110">Structures and classes are similar in the following respects:</span></span>  
  
- <span data-ttu-id="02674-111">Entrambi sono tipi di *contenitori* , ovvero contengono altri tipi come membri.</span><span class="sxs-lookup"><span data-stu-id="02674-111">Both are *container* types, meaning that they contain other types as members.</span></span>  
  
- <span data-ttu-id="02674-112">Entrambi hanno membri, che possono includere costruttori, metodi, proprietà, campi, costanti, enumerazioni, eventi e gestori eventi.</span><span class="sxs-lookup"><span data-stu-id="02674-112">Both have members, which can include constructors, methods, properties, fields, constants, enumerations, events, and event handlers.</span></span> <span data-ttu-id="02674-113">Tuttavia, non confondere questi membri con gli *elementi* dichiarati di una struttura.</span><span class="sxs-lookup"><span data-stu-id="02674-113">However, do not confuse these members with the declared *elements* of a structure.</span></span>  
  
- <span data-ttu-id="02674-114">I membri di entrambi possono avere livelli di accesso personalizzati.</span><span class="sxs-lookup"><span data-stu-id="02674-114">Members of both can have individualized access levels.</span></span> <span data-ttu-id="02674-115">Ad esempio, un membro può essere dichiarato `Public` e un altro `Private` .</span><span class="sxs-lookup"><span data-stu-id="02674-115">For example, one member can be declared `Public` and another `Private`.</span></span>  
  
- <span data-ttu-id="02674-116">Entrambi possono implementare interfacce.</span><span class="sxs-lookup"><span data-stu-id="02674-116">Both can implement interfaces.</span></span>  
  
- <span data-ttu-id="02674-117">Entrambi possono avere costruttori condivisi, con o senza parametri.</span><span class="sxs-lookup"><span data-stu-id="02674-117">Both can have shared constructors, with or without parameters.</span></span>  
  
- <span data-ttu-id="02674-118">Entrambi possono esporre una *proprietà predefinita*, purché la proprietà accetti almeno un parametro.</span><span class="sxs-lookup"><span data-stu-id="02674-118">Both can expose a *default property*, provided that property takes at least one parameter.</span></span>  
  
- <span data-ttu-id="02674-119">Entrambi possono dichiarare e generare eventi ed entrambi possono dichiarare delegati.</span><span class="sxs-lookup"><span data-stu-id="02674-119">Both can declare and raise events, and both can declare delegates.</span></span>  
  
## <a name="differences"></a><span data-ttu-id="02674-120">Differenze</span><span class="sxs-lookup"><span data-stu-id="02674-120">Differences</span></span>  
 <span data-ttu-id="02674-121">Le strutture e le classi differiscono dai seguenti elementi:</span><span class="sxs-lookup"><span data-stu-id="02674-121">Structures and classes differ in the following particulars:</span></span>  
  
- <span data-ttu-id="02674-122">Le strutture sono *tipi valore*; le classi sono *tipi di riferimento*.</span><span class="sxs-lookup"><span data-stu-id="02674-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="02674-123">Una variabile di un tipo di struttura contiene i dati della struttura, anziché contenere un riferimento ai dati come tipo di classe.</span><span class="sxs-lookup"><span data-stu-id="02674-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
- <span data-ttu-id="02674-124">Le strutture usano l'allocazione dello stack; le classi usano l'allocazione heap.</span><span class="sxs-lookup"><span data-stu-id="02674-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
- <span data-ttu-id="02674-125">Tutti gli elementi della struttura sono `Public` per impostazione predefinita. le variabili di classe e le costanti sono per impostazione predefinita `Private` , mentre altri membri della classe sono `Public` per impostazione predefinita.</span><span class="sxs-lookup"><span data-stu-id="02674-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="02674-126">Questo comportamento per i membri della classe garantisce la compatibilità con il sistema Visual Basic 6,0 di impostazioni predefinite.</span><span class="sxs-lookup"><span data-stu-id="02674-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
- <span data-ttu-id="02674-127">Una struttura deve avere almeno una variabile non condivisa o un elemento evento non condiviso, non personalizzato; una classe può essere completamente vuota.</span><span class="sxs-lookup"><span data-stu-id="02674-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
- <span data-ttu-id="02674-128">Gli elementi della struttura non possono essere dichiarati come `Protected` . i membri della classe possono.</span><span class="sxs-lookup"><span data-stu-id="02674-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
- <span data-ttu-id="02674-129">Una procedura di struttura è in grado di gestire gli eventi solo se si tratta di una procedura [condivisa](../../../language-reference/modifiers/shared.md) `Sub` e solo tramite l' [istruzione AddHandler](../../../language-reference/statements/addhandler-statement.md); qualsiasi routine della classe può gestire gli eventi, utilizzando la parola chiave [Handles](../../../language-reference/statements/handles-clause.md) o l' `AddHandler` istruzione.</span><span class="sxs-lookup"><span data-stu-id="02674-129">A structure procedure can handle events only if it is a [Shared](../../../language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="02674-130">Per altre informazioni, vedere [Eventi](../events/index.md).</span><span class="sxs-lookup"><span data-stu-id="02674-130">For more information, see [Events](../events/index.md).</span></span>  
  
- <span data-ttu-id="02674-131">Le dichiarazioni di variabili di struttura non possono specificare inizializzatori o dimensioni iniziali per le matrici; le dichiarazioni di variabili di classe possono.</span><span class="sxs-lookup"><span data-stu-id="02674-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>  
  
- <span data-ttu-id="02674-132">Le strutture ereditano in modo implicito dalla <xref:System.ValueType?displayProperty=nameWithType> classe e non possono ereditare da altri tipi. le classi possono ereditare da qualsiasi classe o classe diversa da <xref:System.ValueType?displayProperty=nameWithType> .</span><span class="sxs-lookup"><span data-stu-id="02674-132">Structures implicitly inherit from the <xref:System.ValueType?displayProperty=nameWithType> class and cannot inherit from any other type; classes can inherit from any class or classes other than <xref:System.ValueType?displayProperty=nameWithType>.</span></span>  
  
- <span data-ttu-id="02674-133">Le strutture non sono ereditabili; le classi sono.</span><span class="sxs-lookup"><span data-stu-id="02674-133">Structures are not inheritable; classes are.</span></span>  
  
- <span data-ttu-id="02674-134">Le strutture non vengono mai terminate, quindi il Common Language Runtime (CLR) non chiama mai il <xref:System.Object.Finalize%2A> metodo su una struttura; le classi vengono terminate dalla Garbage Collector (GC), che chiama <xref:System.Object.Finalize%2A> su una classe quando rileva che non sono presenti riferimenti attivi rimanenti.</span><span class="sxs-lookup"><span data-stu-id="02674-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
- <span data-ttu-id="02674-135">Una struttura non richiede un costruttore. una classe.</span><span class="sxs-lookup"><span data-stu-id="02674-135">A structure does not require a constructor; a class does.</span></span>  
  
- <span data-ttu-id="02674-136">Le strutture possono avere costruttori non condivisi solo se accettano parametri. le classi possono avere con o senza parametri.</span><span class="sxs-lookup"><span data-stu-id="02674-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="02674-137">Ogni struttura ha un costruttore pubblico implicito senza parametri.</span><span class="sxs-lookup"><span data-stu-id="02674-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="02674-138">Questo costruttore inizializza tutti gli elementi dati della struttura sui valori predefiniti.</span><span class="sxs-lookup"><span data-stu-id="02674-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="02674-139">Non è possibile ridefinire questo comportamento.</span><span class="sxs-lookup"><span data-stu-id="02674-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="02674-140">Istanze e variabili</span><span class="sxs-lookup"><span data-stu-id="02674-140">Instances and Variables</span></span>  
 <span data-ttu-id="02674-141">Poiché le strutture sono tipi di valore, ogni variabile di struttura viene associata in modo permanente a una singola istanza della struttura.</span><span class="sxs-lookup"><span data-stu-id="02674-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="02674-142">Tuttavia, le classi sono tipi di riferimento e una variabile oggetto può fare riferimento a diverse istanze di classe in momenti diversi.</span><span class="sxs-lookup"><span data-stu-id="02674-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="02674-143">Questa distinzione influiscono sull'utilizzo di strutture e classi nei modi seguenti:</span><span class="sxs-lookup"><span data-stu-id="02674-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
- <span data-ttu-id="02674-144">**Inizializzazione.**</span><span class="sxs-lookup"><span data-stu-id="02674-144">**Initialization.**</span></span> <span data-ttu-id="02674-145">Una variabile di struttura include implicitamente un'inizializzazione degli elementi usando il costruttore senza parametri della struttura.</span><span class="sxs-lookup"><span data-stu-id="02674-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="02674-146">Pertanto, `Dim s As struct1` equivale a `Dim s As struct1 = New struct1()` .</span><span class="sxs-lookup"><span data-stu-id="02674-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>

- <span data-ttu-id="02674-150">**Assegnazione di Nothing.**</span><span class="sxs-lookup"><span data-stu-id="02674-150">**Assigning Nothing.**</span></span> <span data-ttu-id="02674-151">È possibile assegnare il valore [Nothing](../../../language-reference/nothing.md) a una variabile di struttura, ma l'istanza continua a essere associata alla variabile.</span><span class="sxs-lookup"><span data-stu-id="02674-151">You can assign the value [Nothing](../../../language-reference/nothing.md) to a structure variable, but the instance continues to be associated with the variable.</span></span> <span data-ttu-id="02674-152">È comunque possibile chiamare i relativi metodi e accedere ai relativi elementi dati, anche se gli elementi variabili vengono reinizializzati dall'assegnazione.</span><span class="sxs-lookup"><span data-stu-id="02674-152">You can still call its methods and access its data elements, although variable elements are reinitialized by the assignment.</span></span>  
  
     <span data-ttu-id="02674-153">Al contrario, se si imposta una variabile oggetto su, l'associazione viene `Nothing` annullata da qualsiasi istanza di classe e non è possibile accedere ai membri tramite la variabile finché non viene assegnata un'altra istanza.</span><span class="sxs-lookup"><span data-stu-id="02674-153">In contrast, if you set an object variable to `Nothing`, you dissociate it from any class instance, and you cannot access any members through the variable until you assign another instance to it.</span></span>  
  
- <span data-ttu-id="02674-154">**Più istanze.**</span><span class="sxs-lookup"><span data-stu-id="02674-154">**Multiple Instances.**</span></span> <span data-ttu-id="02674-155">A una variabile oggetto possono essere assegnate istanze di classe diverse in momenti diversi e diverse variabili oggetto possono fare riferimento alla stessa istanza della classe nello stesso momento.</span><span class="sxs-lookup"><span data-stu-id="02674-155">An object variable can have different class instances assigned to it at different times, and several object variables can refer to the same class instance at the same time.</span></span> <span data-ttu-id="02674-156">Le modifiche apportate ai valori dei membri della classe hanno effetto su tali membri quando si accede tramite un'altra variabile che punta alla stessa istanza.</span><span class="sxs-lookup"><span data-stu-id="02674-156">Changes you make to the values of class members affect those members when accessed through another variable pointing to the same instance.</span></span>  
  
     <span data-ttu-id="02674-157">Gli elementi della struttura, tuttavia, sono isolati all'interno della propria istanza.</span><span class="sxs-lookup"><span data-stu-id="02674-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="02674-158">Le modifiche apportate ai valori non vengono riflesse in altre variabili di struttura, neanche in altre istanze della stessa `Structure` dichiarazione.</span><span class="sxs-lookup"><span data-stu-id="02674-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>  
  
- <span data-ttu-id="02674-159">**Uguaglianza.**</span><span class="sxs-lookup"><span data-stu-id="02674-159">**Equality.**</span></span> <span data-ttu-id="02674-160">I test di uguaglianza di due strutture devono essere eseguiti con un test elemento per elemento.</span><span class="sxs-lookup"><span data-stu-id="02674-160">Equality testing of two structures must be performed with an element-by-element test.</span></span> <span data-ttu-id="02674-161">È possibile confrontare due variabili oggetto utilizzando il <xref:System.Object.Equals%2A> metodo.</span><span class="sxs-lookup"><span data-stu-id="02674-161">Two object variables can be compared using the <xref:System.Object.Equals%2A> method.</span></span> <span data-ttu-id="02674-162"><xref:System.Object.Equals%2A>indica se le due variabili puntano alla stessa istanza.</span><span class="sxs-lookup"><span data-stu-id="02674-162"><xref:System.Object.Equals%2A> indicates whether the two variables point to the same instance.</span></span>

<span data-ttu-id="d4207-113">Anche il valore dell'elemento `variable` può essere assegnato in diversi modi.</span><span class="sxs-lookup"><span data-stu-id="d4207-113">For the `variable` element, the value is also assigned in several ways.</span></span> <span data-ttu-id="d4207-114">È possibile assegnarlo tramite il contenuto restituito dall'espressione XPath nell'attributo `select` oppure usando il contenuto del corpo del modello.</span><span class="sxs-lookup"><span data-stu-id="d4207-114">You can assign it by returning content from the XPath expression in the `select` attribute, or by assigning it the contents of the template body.</span></span>

<span data-ttu-id="d4207-115">Da entrambi gli elementi `parameter` e `variable`, se viene assegnato un valore dall'espressione XPath, viene restituito uno dei quattro tipi XPath principali: Boolean, string, number o node set.</span><span class="sxs-lookup"><span data-stu-id="d4207-115">For both the `parameter` and `variable` elements, if a value is assigned by the XPath expression, then one of the four basic XPath types will be returned: Boolean, string, number, or node set.</span></span> <span data-ttu-id="d4207-116">Se il valore viene assegnato usando un corpo del modello non vuoto, verrà restituito un tipo di dati non XPath, ovvero un frammento di albero risultato.</span><span class="sxs-lookup"><span data-stu-id="d4207-116">When the value is given by using a non-empty template body, then a non-XPath data type is returned, and that will be a result tree fragment.</span></span>

<span data-ttu-id="d4207-117">Una query XPath restituisce un tipo di dati che non appartiene a uno dei quattro tipi di oggetti XPath unicamente nel caso in cui una variabile è associata a un frammento <span data-ttu-id="d4207-120">Un frammento di albero risultato si comporta, teoricamente, come un set di nodi con un unico nodo radice,</span><span class="sxs-lookup"><span data-stu-id="d4207-120">A result tree fragment, in concept, behaves like a node set with nothing more than a single root node.</span></span> <span data-ttu-id="d4207-121">mentre gli altri nodi restituiti sono nodi figlio.</span><span class="sxs-lookup"><span data-stu-id="d4207-121">However, the rest of the nodes returned are child nodes.</span></span> <span data-ttu-id="d4207-122">Per visualizzare i nodi figlio a livello di codice, copiare il frammento di albero risultato nell'albero risultato stessa usando l'elemento `<xsl:copy-of>`.</span><span class="sxs-lookup"><span data-stu-id="d4207-122">To programmatically see the child nodes, copy the result tree fragment to the result tree using the `<xsl:copy-of>` element.</span></span> <span data-ttu-id="d4207-123">Questa operazione consente di copiare in sequenza tutti i nodi figlio nell'albero risultato.</span><span class="sxs-lookup"><span data-stu-id="d4207-123">When the copy-of is performed, all the child nodes are also copied to the result tree in sequence.</span></span> <span data-ttu-id="d4207-124">Il frammento di albero risultato non farà parte dell'albero risultato o dell'output della trasformazione finché non viene usato un comando `copy` o `copy-of`.</span><span class="sxs-lookup"><span data-stu-id="d4207-124">Until a `copy` or `copy-of` is used, a result tree fragment is not part of the result tree or the output from the transformation.</span></span>

<span data-ttu-id="d4207-125">Per eseguire un'iterazione sui nodi restituiti in un frammento di albero risultato, usare <xref:System.Xml.XPath.XPathNavigator>.</span><span class="sxs-lookup"><span data-stu-id="d4207-125">To iterate over the returned nodes of a result tree fragment, an <xref:System.Xml.XPath.XPathNavigator> is used.</span></span> <span data-ttu-id="d4207-126">Nel codice di esempio seguente viene illustrata la creazione di un frammento di albero risultato in un foglio di stile, chiamando la funzione con un parametro `fragment` contenente dati XML.</span><span class="sxs-lookup"><span data-stu-id="d4207-126">The following code sample shows how to create a result tree fragment within a style sheet by calling the function with a parameter `fragment`, which contains XML.</span></span>

```xml
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
                xmlns:msxsl="urn:schemas-microsoft-com:xslt"
                xmlns:user="http://www.adventure-works.com"
                version="1.0">
    <xsl:var name="fragment">
        <node1>
            <node2/>
        </node1>
    <xsl:var>

  <msxsl:script language="C#" implements-prefix="user">
    function NodeFragment(XPathNavigator nav)
    {
      if (nav.HasSelection == false)
        XPathNavigator.MoveToNext();
      return;
    }
  </msxsl:script>

    <xsl:template match="/">
            <xsl:value-of select="user:NodeFragment(msxml:node-set($fragment))"/>
    </xsl:template>
</xsl:stylesheet>
```

<span data-ttu-id="d4207-127">Di seguito è riportato un altro esempio in cui viene mostrata una variabile, in RTF (Rich Text Format) e, di conseguenza, un tipo di frammento di albero risultato nodo non convertito in un set di nodi.</span><span class="sxs-lookup"><span data-stu-id="d4207-127">Here is another sample showing a variable, which is in Rich Text Format (RTF), and hence, a type of result tree fragment, that is not converted to a node set.</span></span> <span data-ttu-id="d4207-128">Il frammento viene passato a una funzione script e per esplorare i nodi viene usato il tipo <xref:System.Xml.XPath.XPathNavigator>.</span><span class="sxs-lookup"><span data-stu-id="d4207-128">Instead, it is passed to a script function, and the <xref:System.Xml.XPath.XPathNavigator> is used to navigate over the nodes.</span></span>

```xml
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
        xmlns:msxsl="urn:schemas-microsoft-com:xslt"
        xmlns:user="urn:books"
        exclude-result-prefixes="msxsl">

<xsl:output method="xml" indent="yes" omit-xml-declaration="yes"/>

<xsl:variable name="node-fragment">
    <book>Book1</book>
    <book>Book2</book>
    <book>Book3</book>
    <book>Book4</book>
</xsl:variable>

<msxsl:script implements-prefix="user" language="c#">

<![CDATA[
    string func(XPathNavigator nav)
    {
        bool b = nav.MoveToFirstChild();
        if (b)
            return nav.Value;
        else
            return "Does not exist";
    }

]]>

</msxsl:script>

<xsl:template match="/">
    <first_book>
        <xsl:value-of select="user:func($node-fragment)"/>
    </first_book>
</xsl:template>
```

## <a name="marshal-by-value"></a><span data-ttu-id="ad40c-120">Marshalling in base al valore</span><span class="sxs-lookup"><span data-stu-id="ad40c-120">Marshal by value</span></span>
<span data-ttu-id="ad40c-121">Gli oggetti sono validi solo nel dominio dell'applicazione in cui sono stati creati.</span><span class="sxs-lookup"><span data-stu-id="ad40c-121">Objects are valid only in the application domain where they are created.</span></span> <span data-ttu-id="ad40c-122">Qualsiasi tentativo di passare l'oggetto come parametro o restituirlo come risultato non avrà esito positivo, a meno che l'oggetto non derivi da `MarshalByRefObject` o sia contrassegnato come `Serializable`.</span><span class="sxs-lookup"><span data-stu-id="ad40c-122">Any attempt to pass the object as a parameter or return it as a result will fail unless the object derives from `MarshalByRefObject` or is marked as `Serializable`.</span></span> <span data-ttu-id="ad40c-123">Se l'oggetto è contrassegnato come `Serializable`, l'oggetto sarà serializzato automaticamente, verrà trasportato da un dominio dell'applicazione all'altro e sarà successivamente deserializzato in modo da produrre una copia esatta dell'oggetto nel secondo dominio dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="ad40c-123">If the object is marked as `Serializable`, the object will automatically be serialized, transported from the one application domain to the other, and then deserialized to produce an exact copy of the object in the second application domain.</span></span> <span data-ttu-id="ad40c-124">Tale processo viene generalmente denominato marshalling in base al valore.</span><span class="sxs-lookup"><span data-stu-id="ad40c-124">This process is typically referred to as marshal-by-value.</span></span>

<span data-ttu-id="ad40c-125">Quando un oggetto deriva da `MarshalByRefObject`, viene passato un riferimento all'oggetto da un dominio dell'applicazione a un altro e non l'oggetto stesso.</span><span class="sxs-lookup"><span data-stu-id="ad40c-125">When an object derives from `MarshalByRefObject`, an object reference is passed from one application domain to another, rather than the object itself.</span></span> <span data-ttu-id="ad40c-126">È anche possibile contrassegnare un oggetto che deriva da `MarshalByRefObject` come `Serializable`.</span><span class="sxs-lookup"><span data-stu-id="ad40c-126">You can also mark an object that derives from `MarshalByRefObject` as `Serializable`.</span></span> <span data-ttu-id="ad40c-127">Se questo oggetto viene usato con i sevizi remoti, il formattatore responsabile della serializzazione, preconfigurato con un selettore di surrogati (`SurrogateSelector`), prende il controllo del processo di serializzazione e sostituisce tutti gli oggetti derivati da `MarshalByRefObject` con un proxy.</span><span class="sxs-lookup"><span data-stu-id="ad40c-127">When this object is used with remoting, the formatter responsible for serialization, which has been preconfigured with a surrogate selector (`SurrogateSelector`), takes control of the serialization process, and replaces all objects derived from `MarshalByRefObject` with a proxy.</span></span> <span data-ttu-id="ad40c-128">Senza `SurrogateSelector`, invece, l'architettura della serializzazione segue le regole della serializzazione standard descritte in [Passaggi del processo di serializzazione](steps-in-the-serialization-process.md).</span><span class="sxs-lookup"><span data-stu-id="ad40c-128">Without the `SurrogateSelector` in place, the serialization architecture follows the standard serialization rules described in [Steps in the Serialization Process](steps-in-the-serialization-process.md).</span></span>

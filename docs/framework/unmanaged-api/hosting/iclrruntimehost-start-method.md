# <a name="iclrruntimehoststart-method"></a>Metodo ICLRRuntimeHost::Start
Inizializza il Common Language Runtime (CLR) in un processo.  
  
## <a name="syntax"></a>Sintassi  
  
```cpp  
HRESULT Start();  
```  
  
## <a name="return-value"></a>Valore restituito  
  
|HRESULT|Description|  
|-------------|-----------------|  
|S_OK|`Start`la restituzione è riuscita.|  
|HOST_E_CLRNOTAVAILABLE|CLR non è stato caricato in un processo oppure CLR si trova in uno stato in cui non è possibile eseguire codice gestito o elaborare la chiamata correttamente.|  
|HOST_E_TIMEOUT|Timeout della chiamata.|  
|HOST_E_NOT_OWNER|Il chiamante non è il proprietario del blocco.|  
|HOST_E_ABANDONED|Un evento è stato annullato mentre un thread bloccato o Fiber era in attesa su di esso.|  
|E_FAIL|Si è verificato un errore irreversibile sconosciuto. Se un metodo restituisce E_FAIL, CLR non è più utilizzabile all'interno del processo. Le chiamate successive ai metodi di hosting restituiscono HOST_E_CLRNOTAVAILABLE.|  
  
## <a name="remarks"></a>Osservazioni  
 In molti scenari non è necessario chiamare `Start` , perché il runtime si inizializza automaticamente alla prima richiesta di esecuzione del codice gestito. È tuttavia possibile utilizzare `Start` per specificare esattamente quando il runtime deve essere inizializzato.

 <span data-ttu-id="93e11-108">Il tipo <xref:System.ServiceModel.Description.WsdlExporter?displayProperty=nameWithType> genera metadati Web Services Description Language (WSDL) con allegate espressioni di criteri incapsulate in un'istanza di <xref:System.ServiceModel.Description.MetadataSet>.</span><span class="sxs-lookup"><span data-stu-id="93e11-108">The <xref:System.ServiceModel.Description.WsdlExporter?displayProperty=nameWithType> type generates Web Services Description Language (WSDL) metadata with attached policy expressions encapsulated in a <xref:System.ServiceModel.Description.MetadataSet> instance.</span></span> <span data-ttu-id="93e11-109">È possibile utilizzare un'istanza di <xref:System.ServiceModel.Description.WsdlExporter?displayProperty=nameWithType> per esportare in modo iterativo metadati per oggetti <xref:System.ServiceModel.Description.ContractDescription> e <xref:System.ServiceModel.Description.ServiceEndpoint>.</span><span class="sxs-lookup"><span data-stu-id="93e11-109">You can use a <xref:System.ServiceModel.Description.WsdlExporter?displayProperty=nameWithType> instance to iteratively export metadata for <xref:System.ServiceModel.Description.ContractDescription> objects and <xref:System.ServiceModel.Description.ServiceEndpoint> objects.</span></span> <span data-ttu-id="93e11-110">È inoltre possibile esportare una raccolta di oggetti <xref:System.ServiceModel.Description.ServiceEndpoint> e associarli a un nome del servizio specifico.</span><span class="sxs-lookup"><span data-stu-id="93e11-110">You can also export a collection of <xref:System.ServiceModel.Description.ServiceEndpoint> objects and associate them with a specific service name.</span></span>

### <a name="importing-policies"></a><span data-ttu-id="93e11-124">Importazione di criteri</span><span class="sxs-lookup"><span data-stu-id="93e11-124">Importing Policies</span></span>  
 <span data-ttu-id="93e11-125">Il tipo <xref:System.ServiceModel.Description.WsdlImporter> raccoglie le espressioni di criteri allegate ai soggetti di messaggio, operazione ed endpoint, quindi utilizza le implementazioni <xref:System.ServiceModel.Description.IPolicyImportExtension> nella raccolta <xref:System.ServiceModel.Description.MetadataImporter.PolicyImportExtensions%2A> per importare le espressioni di criteri.</span><span class="sxs-lookup"><span data-stu-id="93e11-125">The <xref:System.ServiceModel.Description.WsdlImporter> type collects the policy expressions attached to the message, operation, and endpoint policy subjects and then uses the <xref:System.ServiceModel.Description.IPolicyImportExtension> implementations in the <xref:System.ServiceModel.Description.MetadataImporter.PolicyImportExtensions%2A> collection to import the policy expressions.</span></span>  
  
 <span data-ttu-id="93e11-126">La logica di importazione del criterio gestisce automaticamente i riferimenti del criterio alle espressioni di criteri nello stesso documento WSDL ed è identificata da un attributo `wsu:Id` o `xml:id`.</span><span class="sxs-lookup"><span data-stu-id="93e11-126">The policy import logic automatically handles policy references to policy expressions in the same WSDL document and is identified with a `wsu:Id` or `xml:id` attribute.</span></span> <span data-ttu-id="93e11-127">La logica di importazione del criterio protegge le applicazioni da riferimenti del criterio circolari limitando la dimensione di un'espressione di criteri a 4096 nodi, in cui un nodo è uno degli elementi seguenti: `wsp:Policy`, `wsp:All`, `wsp:ExactlyOne`, `wsp:policyReference`.</span><span class="sxs-lookup"><span data-stu-id="93e11-127">The policy import logic protects applications against circular policy references by limiting the size of a policy expression to 4096 nodes, where a node is a one of the following elements: `wsp:Policy`, `wsp:All`, `wsp:ExactlyOne`, `wsp:policyReference`.</span></span>  
  
 <span data-ttu-id="93e11-128">La logica di importazione del criterio normalizza inoltre automaticamente le espressioni di criteri.</span><span class="sxs-lookup"><span data-stu-id="93e11-128">The policy import logic also automatically normalizes policy expressions.</span></span> <span data-ttu-id="93e11-129">Espressioni di criteri annidate e l'attributo `wsp:Optional` non sono normalizzati.</span><span class="sxs-lookup"><span data-stu-id="93e11-129">Nested policy expressions and the `wsp:Optional` attribute are not normalized.</span></span> <span data-ttu-id="93e11-130">Il quantità di elaborazione di normalizzazione eseguita è limitata a 4096 passaggi, in cui ogni passaggio produce un'asserzione di criteri, o un elemento figlio di un elemento `wsp:ExactlyOne`.</span><span class="sxs-lookup"><span data-stu-id="93e11-130">The amount of normalization processing done is limited to 4096 steps, where each step yields a policy assertion, or a child element of a `wsp:ExactlyOne` element.</span></span>  
  
 <span data-ttu-id="93e11-131">Il tipo <xref:System.ServiceModel.Description.WsdlImporter> tenta un massimo di 32 combinazioni di alternative di criteri allegate ai diversi soggetti di criterio WSDL.</span><span class="sxs-lookup"><span data-stu-id="93e11-131">The <xref:System.ServiceModel.Description.WsdlImporter> type tries up to 32 combinations of policy alternatives attached to the different WSDL policy subjects.</span></span> <span data-ttu-id="93e11-132">Se nessuna combinazione viene importata correttamente, viene utilizzata la prima combinazione per costruire un'associazione personalizzata parziale.</span><span class="sxs-lookup"><span data-stu-id="93e11-132">If no combination imports cleanly, the first combination is used to construct a partial custom binding.</span></span>  
  
## <a name="error-handling"></a><span data-ttu-id="93e11-133">Gestione degli errori</span><span class="sxs-lookup"><span data-stu-id="93e11-133">Error Handling</span></span>

 <span data-ttu-id="93e11-135">Il tipo <xref:System.ServiceModel.Description.WsdlImporter> di norma genera un'eccezione per un'eccezione intercettata durante il processo di importazione e aggiunge un errore corrispondente alla propria proprietà `Errors`.</span><span class="sxs-lookup"><span data-stu-id="93e11-135">The <xref:System.ServiceModel.Description.WsdlImporter> type generally throws an exception for an exception caught during the import process and adds a corresponding error to its `Errors` property.</span></span> <span data-ttu-id="93e11-136">I metodi <xref:System.ServiceModel.Description.WsdlImporter.ImportAllContracts%2A>, <xref:System.ServiceModel.Description.WsdlImporter.ImportAllBindings%2A>, <xref:System.ServiceModel.Description.WsdlImporter.ImportAllEndpoints%2A> e <xref:System.ServiceModel.Description.WsdlImporter.ImportEndpoints%2A>, tuttavia, non generano queste eccezioni, pertanto è necessario controllare la proprietà `Errors` per stabilire se si sono verificati problemi durante la chiamata di questi metodi.</span><span class="sxs-lookup"><span data-stu-id="93e11-136">The <xref:System.ServiceModel.Description.WsdlImporter.ImportAllContracts%2A>, <xref:System.ServiceModel.Description.WsdlImporter.ImportAllBindings%2A>, <xref:System.ServiceModel.Description.WsdlImporter.ImportAllEndpoints%2A>, and <xref:System.ServiceModel.Description.WsdlImporter.ImportEndpoints%2A> methods, however, do not throw these exceptions, so you must check the `Errors` property to determine if any issues occurred when calling these methods.</span></span>

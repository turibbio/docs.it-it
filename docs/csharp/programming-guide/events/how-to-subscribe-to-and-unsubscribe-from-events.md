### <a name="to-subscribe-to-events-programmatically"></a><span data-ttu-id="c4cef-114">Per sottoscrivere gli eventi a livello di codice</span><span class="sxs-lookup"><span data-stu-id="c4cef-114">To subscribe to events programmatically</span></span>  
  
1. <span data-ttu-id="c4cef-115">Definire un metodo del gestore eventi la cui firma corrisponda alla firma del delegato per l'evento.</span><span class="sxs-lookup"><span data-stu-id="c4cef-115">Define an event handler method whose signature matches the delegate signature for the event.</span></span> <span data-ttu-id="c4cef-116">Se ad esempio l'evento è basato sul tipo di delegato <xref:System.EventHandler>, il codice riportato di seguito rappresenta lo stub del metodo:</span><span class="sxs-lookup"><span data-stu-id="c4cef-116">For example, if the event is based on the <xref:System.EventHandler> delegate type, the following code represents the method stub:</span></span>  
  
    ```csharp

    {  
       // Do something useful here.  
    }  
    ```  
  
2. <span data-ttu-id="c4cef-117">Usare l'operatore di assegnazione di addizione (`+=`) per associare un gestore all'evento.</span><span class="sxs-lookup"><span data-stu-id="c4cef-117">Use the addition assignment operator (`+=`) to attach an event handler to the event.</span></span> <span data-ttu-id="c4cef-118">Nell'esempio seguente si supponga che a un oggetto denominato `publisher` sia associato un evento denominato `RaiseCustomEvent`.</span><span class="sxs-lookup"><span data-stu-id="c4cef-118">In the following example, assume that an object named `publisher` has an event named `RaiseCustomEvent`.</span></span> <span data-ttu-id="c4cef-119">Si noti che per la classe subscriber è necessario un riferimento alla classe publisher per sottoscrivere gli eventi corrispondenti.</span><span class="sxs-lookup"><span data-stu-id="c4cef-119">Note that the subscriber class needs a reference to the publisher class in order to subscribe to its events.</span></span>  
  
    ```csharp

    ```  
  
     <span data-ttu-id="c4cef-120">Si noti che la sintassi precedente è nuova in C# 2.0.</span><span class="sxs-lookup"><span data-stu-id="c4cef-120">Note that the previous syntax is new in C# 2.0.</span></span> <span data-ttu-id="c4cef-121">Equivale esattamente alla sintassi di C# 1.0, in cui è necessario creare in modo esplicito il delegato incapsulante tramite la parola chiave `new`:</span><span class="sxs-lookup"><span data-stu-id="c4cef-121">It is exactly equivalent to the C# 1.0 syntax in which the encapsulating delegate must be explicitly created by using the `new` keyword:</span></span>  
  
    ```csharp
    publisher.RaiseCustomEvent += new CustomEventHandler(HandleCustomEvent);  
    ```  
  
     <span data-ttu-id="c4cef-122">È anche possibile usare [un'espressione lambda](../statements-expressions-operators/lambda-expressions.md) per specificare un gestore eventi:You can also use a lambda expression to specify an event handler:</span><span class="sxs-lookup"><span data-stu-id="c4cef-122">You can also use a [lambda expression](../statements-expressions-operators/lambda-expressions.md) to specify an event handler:</span></span>

    ```csharp
                MessageBox.Show(((MouseEventArgs)e).Location.ToString());
    ```  

### <a name="to-subscribe-to-events-by-using-an-anonymous-method"></a><span data-ttu-id="c4cef-123">Per sottoscrivere gli eventi usando un metodo anonimo</span><span class="sxs-lookup"><span data-stu-id="c4cef-123">To subscribe to events by using an anonymous method</span></span>  
  
- <span data-ttu-id="c4cef-124">Se non è necessario annullare la sottoscrizione di un evento in un secondo momento, è possibile usare l'operatore di assegnazione di addizione (`+=`) per associare un metodo anonimo all'evento.</span><span class="sxs-lookup"><span data-stu-id="c4cef-124">If you will not have to unsubscribe to an event later, you can use the addition assignment operator (`+=`) to attach an anonymous method to the event.</span></span> <span data-ttu-id="c4cef-125">Nell'esempio seguente si supponga che a un oggetto denominato `publisher` sia associato un evento denominato `RaiseCustomEvent` e che sia stata definita una classe `CustomEventArgs` con informazioni specializzate sull'evento.</span><span class="sxs-lookup"><span data-stu-id="c4cef-125">In the following example, assume that an object named `publisher` has an event named `RaiseCustomEvent` and that a `CustomEventArgs` class has also been defined to carry some kind of specialized event information.</span></span> <span data-ttu-id="c4cef-126">Si noti che per la classe subscriber è necessario un riferimento alla classe `publisher` per sottoscrivere gli eventi corrispondenti.</span><span class="sxs-lookup"><span data-stu-id="c4cef-126">Note that the subscriber class needs a reference to `publisher` in order to subscribe to its events.</span></span>  
  

     <span data-ttu-id="c4cef-127">È importante notare che non si può annullare facilmente la sottoscrizione di un evento se per la sottoscrizione è stata usata una funzione anonima.</span><span class="sxs-lookup"><span data-stu-id="c4cef-127">It is important to notice that you cannot easily unsubscribe from an event if you used an anonymous function to subscribe to it.</span></span> <span data-ttu-id="c4cef-128">Per annullare la sottoscrizione in questo scenario, è necessario tornare al codice in cui è stato sottoscritto l'evento, archiviare il metodo anonimo in una variabile del delegato e quindi aggiungere il delegato all'evento.</span><span class="sxs-lookup"><span data-stu-id="c4cef-128">To unsubscribe in this scenario, it is necessary to go back to the code where you subscribe to the event, store the anonymous method in a delegate variable, and then add the delegate to the event.</span></span> <span data-ttu-id="c4cef-129">In generale è consigliabile non usare funzioni anonime per sottoscrivere eventi se si prevede di dover annullare la sottoscrizione all'evento in un punto successivo nel codice.</span><span class="sxs-lookup"><span data-stu-id="c4cef-129">In general, we recommend that you do not use anonymous functions to subscribe to events if you will have to unsubscribe from the event at some later point in your code.</span></span> <span data-ttu-id="c4cef-130">Per altre informazioni sulle funzioni anonime, vedere [Funzioni anonime](../statements-expressions-operators/anonymous-functions.md).</span><span class="sxs-lookup"><span data-stu-id="c4cef-130">For more information about anonymous functions, see [Anonymous Functions](../statements-expressions-operators/anonymous-functions.md).</span></span>  
  
## <a name="unsubscribing"></a><span data-ttu-id="c4cef-131">Annullamento della sottoscrizione</span><span class="sxs-lookup"><span data-stu-id="c4cef-131">Unsubscribing</span></span>  
 <span data-ttu-id="c4cef-132">Per evitare che il gestore eventi venga chiamato al momento della generazione dell'evento, annullare la sottoscrizione all'evento stesso.</span><span class="sxs-lookup"><span data-stu-id="c4cef-132">To prevent your event handler from being invoked when the event is raised, unsubscribe from the event.</span></span> <span data-ttu-id="c4cef-133">Per evitare di perdere risorse, è necessario annullare la sottoscrizione agli eventi prima di eliminare un oggetto sottoscrittore.</span><span class="sxs-lookup"><span data-stu-id="c4cef-133">In order to prevent resource leaks, you should unsubscribe from events before you dispose of a subscriber object.</span></span> <span data-ttu-id="c4cef-134">Finché non si annulla la sottoscrizione di un evento, il delegato multicast sottostante all'evento nell'oggetto publisher contiene un riferimento al delegato che incapsula il gestore eventi del sottoscrittore.</span><span class="sxs-lookup"><span data-stu-id="c4cef-134">Until you unsubscribe from an event, the multicast delegate that underlies the event in the publishing object has a reference to the delegate that encapsulates the subscriber's event handler.</span></span> <span data-ttu-id="c4cef-135">Finché l'oggetto publisher include tale riferimento, l'oggetto subscriber non verrà eliminato dal processo di Garbage Collection.</span><span class="sxs-lookup"><span data-stu-id="c4cef-135">As long as the publishing object holds that reference, garbage collection will not delete your subscriber object.</span></span>  
  
#### <a name="to-unsubscribe-from-an-event"></a><span data-ttu-id="c4cef-136">Per annullare la sottoscrizione di un evento</span><span class="sxs-lookup"><span data-stu-id="c4cef-136">To unsubscribe from an event</span></span>

     <span data-ttu-id="c4cef-138">Quando tutti i sottoscrittori hanno annullato la sottoscrizione a un evento, l'istanza dell'evento nella classe publisher viene impostata su `null`.</span><span class="sxs-lookup"><span data-stu-id="c4cef-138">When all subscribers have unsubscribed from an event, the event instance in the publisher class is set to `null`.</span></span>

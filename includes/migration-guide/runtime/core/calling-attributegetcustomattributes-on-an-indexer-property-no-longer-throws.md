### <a name="calling-attributegetcustomattributes-on-an-indexer-property-no-longer-throws-ambiguousmatchexception-if-the-ambiguity-can-be-resolved-by-indexs-type"></a>La chiamata di Attribute.GetCustomAttributes su una proprietà indicizzatore non genera più AmbiguousMatchException se l'ambiguità può essere risolta dal tipo di indice

#### <a name="details"></a>Dettagli

Prima di .NET Framework 4.6, la chiamata di <code>GetCustomAttribute(s)</code> su una proprietà indicizzatore diversa da un'altra proprietà solo per il tipo di indice genera una eccezione <xref:System.Reflection.AmbiguousMatchException?displayProperty=fullName>. A partire da .NET Framework 4.6, gli attributi della proprietà verranno restituiti correttamente.

#### <a name="suggestion"></a>Suggerimento

Tenere presente che GetCustomAttribute funzionerà più frequentemente. Se un'app dipende da <xref:System.Reflection.AmbiguousMatchException?displayProperty=fullName> nelle versioni precedenti, è ora necessario usare la reflection per cercare in modo esplicito più indicizzatori.

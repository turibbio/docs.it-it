### <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a><span data-ttu-id="75f6b-101">I metodi CompareGreaterThan SSE e SSE2 gestiscono correttamente gli input NaN</span><span class="sxs-lookup"><span data-stu-id="75f6b-101">SSE and SSE2 CompareGreaterThan methods properly handle NaN inputs</span></span>

<span data-ttu-id="75f6b-102">I <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> metodi e seguenti <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> sono stati corretti per gestire correttamente gli `NaN` input e corrispondono al comportamento hardware dei metodi equivalenti nella <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> classe:</span><span class="sxs-lookup"><span data-stu-id="75f6b-102">The following <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> and <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> methods have been fixed to properly handle `NaN` inputs and match the hardware behavior of the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> class:</span></span>

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

#### <a name="change-description"></a><span data-ttu-id="75f6b-103">Descrizione modifica:</span><span class="sxs-lookup"><span data-stu-id="75f6b-103">Change description</span></span>

<span data-ttu-id="75f6b-104">In precedenza, `NaN` gli input per i <xref:System.Runtime.Intrinsics.X86.Sse> metodi e elencati <xref:System.Runtime.Intrinsics.X86.Sse2> restituivano un risultato errato.</span><span class="sxs-lookup"><span data-stu-id="75f6b-104">Previously, `NaN` inputs to the listed <xref:System.Runtime.Intrinsics.X86.Sse> and <xref:System.Runtime.Intrinsics.X86.Sse2> methods returned an incorrect result.</span></span> <span data-ttu-id="75f6b-105">Anche il risultato è diverso dal risultato generato dal metodo corrispondente nella <xref:System.Runtime.Intrinsics.X86.Avx> classe.</span><span class="sxs-lookup"><span data-stu-id="75f6b-105">The result also differed from the result generated by the corresponding method in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="75f6b-106">A partire da .NET 5,0, questi metodi gestiscono correttamente gli `NaN` input e restituiscono gli stessi risultati dei metodi corrispondenti nella <xref:System.Runtime.Intrinsics.X86.Avx> classe.</span><span class="sxs-lookup"><span data-stu-id="75f6b-106">Starting in .NET 5.0, these methods correctly handle `NaN` inputs and return the same results as the corresponding methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="75f6b-107">Le architetture standard di settore Streaming SIMD Extensions (SSE) e Streaming SIMD Extensions 2 (SSE2) (ISA) non forniscono supporto hardware diretto per questi metodi di confronto, quindi sono implementate nel software.</span><span class="sxs-lookup"><span data-stu-id="75f6b-107">The Streaming SIMD Extensions (SSE) and Streaming SIMD Extensions 2 (SSE2) industry standard architectures (ISAs) don't provide direct hardware support for these comparison methods, so they're implemented in software.</span></span> <span data-ttu-id="75f6b-108">In precedenza, i metodi venivano implementati in modo non corretto e gli input venivano gestiti in modo errato `NaN` .</span><span class="sxs-lookup"><span data-stu-id="75f6b-108">Previously, the methods were improperly implemented, and they incorrectly handled `NaN` inputs.</span></span> <span data-ttu-id="75f6b-109">Per il codice trasferito da nativo, il comportamento errato può presentare bug.</span><span class="sxs-lookup"><span data-stu-id="75f6b-109">For code ported from native, the incorrect behavior may introduce bugs.</span></span> <span data-ttu-id="75f6b-110">Per un percorso di codice a 256 bit, i metodi possono anche produrre risultati diversi nei metodi equivalenti della <xref:System.Runtime.Intrinsics.X86.Avx> classe.</span><span class="sxs-lookup"><span data-stu-id="75f6b-110">For a 256-bit code path, the methods can also produce different results to the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="75f6b-111">Come esempio del modo in cui i metodi non erano corretti in precedenza, è possibile implementare `CompareNotGreaterThan(x,y)` come `CompareLessThanOrEqual(x,y)` per i valori integer regolari.</span><span class="sxs-lookup"><span data-stu-id="75f6b-111">As an example of how the methods were previously incorrect, you can implement `CompareNotGreaterThan(x,y)` as `CompareLessThanOrEqual(x,y)` for regular integers.</span></span> <span data-ttu-id="75f6b-112">Tuttavia, per `NaN` gli input, la logica calcola il risultato errato.</span><span class="sxs-lookup"><span data-stu-id="75f6b-112">However, for `NaN` inputs, that logic computes the wrong result.</span></span> <span data-ttu-id="75f6b-113">Al contrario, `CompareNotLessThan(y,x)` l'utilizzo di confronta correttamente i numeri *e* prende `NaN` in considerazione gli input.</span><span class="sxs-lookup"><span data-stu-id="75f6b-113">Instead, using `CompareNotLessThan(y,x)` compares the numbers correctly *and* takes `NaN` inputs into consideration.</span></span>
